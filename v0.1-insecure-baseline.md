# v0.1 – Intentionally Insecure Baseline

## Purpose
This version establishes a deliberately insecure Linux server baseline for learning,
testing, and documentation purposes. No hardening has been applied.

---

## System Overview
- Hostname: k1dz3r0-A320M-S2H
- OS: Ubuntu Linux
- Environment: Bare metal
- Network: Private LAN (192.168.12.227/24)

---

## Known Insecure Conditions

### 1. SSH Password Authentication Enabled
- SSH allows password-based login
- No rate limiting or MFA
- Susceptible to brute-force attacks

### 2. Web Server Running With Default Configuration
- Apache installed with default settings
- No TLS (HTTP only)
- Default site exposed

### 3. Sensitive File Publicly Accessible
- Plaintext credentials file exposed via web root
- Example: `/credentials.txt`
- Represents common misconfiguration risk

### 4. Firewall Not Hardened
- No restrictive firewall rules applied
- Services exposed without filtering

### 5. Logs Are Local Only
- Logs exist but are not forwarded or monitored
- No alerting or intrusion detection

---

## Security Status
⚠️ **INTENTIONALLY INSECURE – DO NOT USE IN PRODUCTION**

This system is designed for learning and attack-surface exploration.
